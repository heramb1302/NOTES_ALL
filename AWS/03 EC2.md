
# EC2 related services:

## **1. AWS Lightsail**

_The "Easy Button" for Cloud Hosting._

- **What it is:** A simplified, all-in-one virtual private server (VPS) solution.
    
- **Best for:** Small websites (WordPress, blogs), simple web apps, and development environments.
    
- **Key Feature:** Offers **predictable, fixed monthly pricing** that includes instances, storage, and data transfer.
    
- **Difference from EC2:** Unlike EC2, which is complex and billed per second, Lightsail is designed for users who don't want to configure VPCs or complex networking manually.
    

## **2. AWS Lambda**

_The "Serverless" Champion._

- **What it is:** A Function-as-a-Service (FaaS) that lets you run code without provisioning or managing any servers.
    
- **Best for:** Real-time file processing, backend APIs, and automation tasks.
    
- **Key Feature:** **Automatic Scaling.** It only runs when triggered and scales from zero to thousands of concurrent executions instantly.
    
- **Pricing:** You only pay for the compute time you consume (per millisecond). If your code isn't running, you pay $0$.
    

## **3. AWS Elastic Beanstalk**

_The Managed "Platform as a Service" (PaaS)._

- **What it is:** An orchestration service for deploying applications. You upload your code, and Beanstalk handles the deployment, capacity provisioning, and load balancing.
    
- **Best for:** Developers who want to deploy Java, Python, Node.js, or .NET apps quickly without worrying about the underlying infrastructure.
    
- **Key Feature:** **Full Control.** While it automates the setup, it uses standard AWS resources (like EC2 and Auto Scaling), which you can still go in and manually tweak if needed.
    

## **4. Amazon Elastic Container Service (ECS)**

_The "Container Manager."_

- **What it is:** A highly scalable, high-performance container orchestration service that supports Docker containers.
    
- **Best for:** Microservices architectures and applications that need to be packaged into containers for consistency across different environments.
    
- **Key Feature:** **Deep AWS Integration.** It integrates natively with other services like Route 53, IAM, and CloudWatch.
    
- **Deployment Options:** Can run on top of EC2 instances (which you manage) or using **AWS Fargate** (a serverless engine for containers).
  
  
# Steps Create EC2 instance:
## **Step 1: Navigate to EC2 Dashboard**

1. Log into the **AWS Management Console**.
    
2. In the top search bar, type **EC2** and select the service.
    
3. In the top-right corner, ensure you have the correct **Region** selected (e.g., _Mumbai_ if you are in Maharashtra).
    
4. Click the orange **Launch instance** button.
   

## **Step 2: Name and AMI (OS Selection)**

1. **Name:** Give your server a name (e.g., `My-Web-Server`).
    
2. **Application and OS Images (AMI):** This is your operating system.
    
    - _Recommendation:_ Select **Amazon Linux 2023** or **Ubuntu**.
        
    - Look for the **"Free tier eligible"** label to avoid charges.

## **Step 3: Instance Type and Key Pair**

1. **Instance type:** Choose **t2.micro** (or **t3.micro** depending on your region). Both are usually free-tier eligible and provide 1 vCPU and 1 GB of RAM.
    
2. **Key pair (login):** This is your "physical" key to access the server.
    
    - Click **Create new key pair**.
        
    - **Key pair name:** e.g., `my-laptop-key`.
        
    - **Private key file format:** Choose `.pem` (for OpenSSH/Mac/Linux/Windows Terminal) or `.ppk` (for PuTTY).
        
    - **Download and Save:** Keep this file safe; AWS will not let you download it again!

## **Step 4: Network Settings**

1. **VPC and Subnet:** Leave these as **Default** for now.
    
2. **Auto-assign Public IP:** Ensure this is set to **Enable** so you can reach the server from the internet.
    
3. **Firewall (Security Groups):** * Select **Create security group**.
    
    - **Allow SSH traffic from:** Change this from `Anywhere` to **My IP** for better security.
        
    - Check **Allow HTTP traffic from the internet** if you plan to host a website.

## **Step 5: Configure Storage and Launch**

1. **Configure storage:** The default is usually **8 GiB** or **30 GiB** (gp3). This is plenty for a basic server.
    
2. **Review:** Look at the "Summary" panel on the right.
    
3. Click **Launch instance**.

## **Step 6: Accessing Your Instance**

Once the instance state says **"Running"**:

1. Select the instance and click **Connect**.
    
2. **Option A (Easiest):** Use **EC2 Instance Connect** to open a terminal directly in your browser.
