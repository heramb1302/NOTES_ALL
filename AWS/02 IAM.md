## **Step 1: Create the IAM Group**

Groups make it easier to manage permissions for multiple users at once.

1. Log in to the **AWS Management Console**.
    
2. Search for **IAM** in the services search bar and select it.
    
3. In the left navigation pane, click on **User groups**.
    
4. Click the **Create group** button.
    
5. **User group name:** Give it a descriptive name (e.g., `Network-Admin-Group`).
    
6. **Attach permissions policies:** In the search box, find and check the boxes for:
    
    - `AmazonEC2FullAccess`
        
    - `AmazonVPCFullAccess`
        
7. Click **Create group**.
   
   
## **Step 2: Create the IAM User**

Now, create the specific identity that will use those permissions.

1. In the IAM dashboard left pane, click on **Users**.
    
2. Click **Create user**.
    
3. **User details:** Enter a name (e.g., `dev-user-01`).
    
4. **Console access (Optional):** If this user needs to log in to the website, check **Provide user access to the AWS Management Console**.
    
    - Select **I want to create an IAM user**.
        
    - Set a password (Autogenerated or Custom).
        
5. Click **Next**.
   
   
   
## **Step 3: Assign User to the Group**

This is where the user "inherits" the EC2 and VPC permissions you set up earlier.

1. On the **Set permissions** page, select **Add user to group**.
    
2. Find the group you created in Step 1 (e.g., `Network-Admin-Group`) and check the box next to it.
    
3. Click **Next**.
   
   
## **Step 4: Review and Finalize**

1. Review the user details and the group membership to ensure `AmazonEC2FullAccess` and `AmazonVPCFullAccess` are listed under the group's attached policies.
    
2. Click **Create user**.
    
3. **Important:** On the final screen, click **Download .csv** or copy the **Console sign-in URL**, **Username**, and **Password**.
    
    > **Note:** This is the only time you will see the password. If you miss this, you'll have to reset it later.
