## **Step 1: Create Your Website Files**

Before touching AWS, ensure you have your website ready on your computer.

1. Create a file named `index.html` (your home page).
    
2. (Optional) Create a file named `error.html` (to show if a page is not found).

## **Step 2: Create and Configure the S3 Bucket**

1. Go to the **S3 Console** and click **Create bucket**.
    
2. **Bucket name:** Use a unique name (if you have a domain like `mywebsite.com`, name the bucket exactly that).
    
3. **Region:** Choose the region closest to your users (e.g., **ap-south-1** for Mumbai).
    
4. **Object Ownership:** Select **ACLs disabled** (Recommended).
    
5. **Block Public Access:** **Uncheck** the "Block all public access" box (as we did in the previous step).
    
6. Click **Create bucket**.

## **Step 3: Enable Static Website Hosting**

This tells S3 to treat the bucket like a web server instead of just a storage folder.

1. Click on your bucket name > **Properties** tab.
    
2. Scroll to the very bottom to **Static website hosting** and click **Edit**.
    
3. Select **Enable**.
    
4. **Hosting type:** Choose **Host a static website**.
    
5. **Index document:** Type `index.html`.
    
6. **Error document:** Type `error.html`.
    
7. Click **Save changes**.

## **Step 4: Upload Your Files**

1. Go to the **Objects** tab of your bucket.
    
2. Click **Upload** and drag your `index.html` and any folders (CSS/Images) into the bucket.
    
3. Click **Upload** at the bottom.

## **Step 5: Apply the Public Permissions**

Even though hosting is enabled, the files are still "private" by default. You must apply the policy we generated earlier.

1. Go to the **Permissions** tab > **Bucket policy** > **Edit**.
    
2. Paste this policy (replace `your-bucket-name` with yours):

```
{
    "Version": "2012-10-17",
    "Statement": [
        {
            "Sid": "PublicReadGetObject",
            "Effect": "Allow",
            "Principal": "*",
            "Action": "s3:GetObject",
            "Resource": "arn:aws:s3:::your-bucket-name/*"
        }
    ]
}
```


## **Step 6: Test Your Website**

1. Go back to the **Properties** tab.
    
2. Scroll back down to **Static website hosting**.
    
3. Under **Bucket website endpoint**, you will see a URL (e.g., `http://your-bucket-name.s3-website.ap-south-1.amazonaws.com`).
    
4. Click the link. Your website should now be live!
